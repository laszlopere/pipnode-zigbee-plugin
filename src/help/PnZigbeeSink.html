Zigbee Sink

A Zigbee2MQTT-flavoured MQTT sink — the publish-side companion to the Zigbee Source. It is a specialised MQTT Sink that sits with the rest of the Zigbee palette (magenta body, paper-plane glyph) so a Source → filter → Sink flow reads as a single Zigbee unit on the worksheet.

For now the node only changes its appearance: it publishes exactly what the base MQTT Sink does — each inbound message to its envelope topic, through the configured broker — so all of the base node's configuration (broker profile, topic / payload template, retain, QoS, auth) applies unchanged. See the MQTT Sink help for those settings.

A Z2M-flavoured default (publishing to zigbee2mqtt/<friendly_name>/set with a Z2M-shaped payload) may be layered on in a later revision; until then, stamp the command topic upstream — for example with a Zigbee Relay Command node, which already emits the .../set topic and {"state":"ON"|"OFF"} payload this sink will forward verbatim.
